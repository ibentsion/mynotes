<objective>
Extend `src/train_ctc.py` to (a) accept `--rnn_hidden`, `--num_layers`, and `--params` CLI flags
and (b) expose a reusable `run_training(args, on_epoch_end=...) -> float` helper that the
Phase 5 tuner (plan 03) can call in-process for Optuna pruning. The CLI behaviour stays
backward-compatible: omitting the new flags reproduces today's training run exactly.

Purpose: Without these CLI extensions, Optuna cannot vary architecture (D-02) or feed back
winning configs (D-10). Without `run_training()`, tune.py would need a subprocess-per-trial
model that breaks Optuna pruning across process boundaries (RESEARCH.md Open Question 2,
Pitfall 7).

Output: Modified `src/train_ctc.py` with three new CLI flags and one extracted helper, plus
new tests in `tests/test_train_ctc.py` covering flag wiring, JSON round-trip, and the helper's
callback contract.
</objective>

    on_epoch_end: Callable[[int, float], None] | None = None,

    """Run the full training loop. Returns best_val_cer.

    on_epoch_end(epoch, val_cer) is called after every validation pass when provided.
