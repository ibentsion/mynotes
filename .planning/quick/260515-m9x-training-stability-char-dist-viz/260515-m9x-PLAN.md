  <action>
    **Analysis** (codified in a comment, not just prose):

    Blank-like characters in Hebrew OCR context:
    - CTC blank (index 0): synthetic token. Current bias=-2.0. Post-training blank_frac
      of 0.94-0.99 shows the model LEARNS to prefer blank despite the init — this is a
      training phenomenon (too little data, gradient collapse), not an init problem.
      The right fix is AdamW + LR schedule (commits 2+3). A stronger initial bias (-3.0)
      reduces early collapse risk but won't fix trained blank preference.
    - Space character ' ': a REAL character representing word breaks. Should NOT be
      suppressed — it will naturally appear when words are separated in labels.
    - Niqqud / combining diacritics (Unicode category Mn): if present in charset, they
      are very small marks the CNN sees. No separate bias needed; the LR schedule and
      more data (H) are the correct fixes.
    - Conclusion: only blank index 0 needs bias. Make magnitude configurable to allow
      experimenting with -3.0 or -4.0 without a code change.

    Add to `_build_parser()`:

    ```python
    def _report_char_distribution(
        logger: Any,
        charset: list[str],
        labels: list[str],
    ) -> None:
        """Log character frequency bar chart + table to ClearML at training start.

        Shows which chars are rare vs dominant — informs bias and weighting decisions.
        """
        import matplotlib.pyplot as plt  # noqa: PLC0415
        import unicodedata  # noqa: PLC0415

        counts: dict[str, int] = {}
        for lab in labels:
            for ch in unicodedata.normalize("NFC", lab):
                counts[ch] = counts.get(ch, 0) + 1

        # sort by frequency descending
        sorted_chars = sorted(charset, key=lambda c: -counts.get(c, 0))
        freqs = [counts.get(c, 0) for c in sorted_chars]
        total = max(sum(freqs), 1)
        pcts = [100.0 * f / total for f in freqs]

        # horizontal bar chart so Hebrew glyphs read left→right on y-axis
        fig, ax = plt.subplots(figsize=(8, max(4, len(sorted_chars) * 0.3)))
        y = range(len(sorted_chars))
        ax.barh(list(y), pcts, color="steelblue")
        ax.set_yticks(list(y))
        ax.set_yticklabels(sorted_chars, fontsize=8)
        ax.set_xlabel("% of all characters")
        ax.set_title(f"Character distribution ({total} chars, {len(charset)} unique)")
        ax.invert_yaxis()  # most frequent at top
        plt.tight_layout()
        logger.report_matplotlib_figure(
            title="char_distribution",
            series="labeled_set",
            iteration=0,
            figure=fig,
            report_image=True,
            report_interactive=False,
        )
        plt.close(fig)

        # also log as a table for exact counts
        import pandas as pd  # noqa: PLC0415
        table = pd.DataFrame({
            "char": sorted_chars,
            "count": freqs,
            "pct": [f"{p:.1f}%" for p in pcts],
        })
        logger.report_table(
            title="char_distribution",
            series="table",
            iteration=0,
            table_plot=table,
        )
    ```
