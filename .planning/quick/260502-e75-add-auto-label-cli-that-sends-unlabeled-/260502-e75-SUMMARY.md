# Quick Task 260502-e75: Add Auto-Label CLI Summary

One-liner: OpenAI vision batch-labeler that reads `data/manifest.csv`, encodes each unlabeled crop as base64 PNG, transcribes Hebrew handwriting via `gpt-4o-mini`, and writes labels back atomically after each crop.

## What Was Built

### src/auto_label.py (new)

CLI entry point `main()` with four flags:

| Flag | Default | Purpose |
|------|---------|---------|
| `--manifest` | `data/manifest.csv` | Path to manifest CSV |
| `--model` | `gpt-4o-mini` | OpenAI model (`gpt-4o-mini` or `gpt-4o`) |
| `--dry-run` | off | Count unlabeled without making API calls |
| `--limit N` | off | Cap labeling run to N crops |

Key behaviors:
- Reads manifest with string dtypes for `label`, `notes`, `flag_reasons` columns
- Validates column order against `MANIFEST_COLUMNS` before proceeding
- Requires `OPENAI_API_KEY` in env (exits 2 if missing)
- Resolves relative `crop_path` values relative to manifest's parent directory
- Per-crop exception handling: logs `[WARN]` to stderr, leaves row `status=unlabeled`, continues
- Atomic write after each success: `tempfile.mkstemp` + `os.replace` (no partial writes)
- Prints `[N/total]` progress per crop to stdout
- Catches `KeyboardInterrupt` at top level: prints "Interrupted — progress saved." and exits 130

### pyproject.toml changes

- Added `"openai>=1.0"` to `[project] dependencies`
- Added `auto-label = "src.auto_label:main"` to `[project.scripts]`

## Command Examples

Dry run (no API calls, just count):
```
uv run auto-label --dry-run
```

Label up to 10 crops with the default model:
```
OPENAI_API_KEY=sk-... uv run auto-label --limit 10
```

Label all unlabeled crops with gpt-4o:
```
OPENAI_API_KEY=sk-... uv run auto-label --model gpt-4o
```

## Deferred Follow-ups

Items deliberately out of scope for this task:

1. **Rate-limit handling (429 retry)**: Current implementation propagates `openai.RateLimitError` as a per-crop warning. A future quick task should add exponential backoff with `tenacity` or the `openai` library's built-in retry config.
2. **Batch API**: OpenAI offers an async batch endpoint at 50% cost. Deferred until the labeled dataset is large enough to warrant batch runs.
3. **Confidence / uncertainty signal**: `gpt-4o-mini` returns text only; flagging low-confidence outputs (e.g., empty or very short responses) for human review is not implemented.
