Make two edits inside `main()`:

1. Resolve the manifest from the ClearML dataset when running on the agent. Insert
   this block AFTER `task = init_task(...)` (currently line 319) and BEFORE the
   manifest existence check (currently line 321):

   ```python
   # Resolve manifest from dataset when not available locally (remote agent path)
   if args.dataset_id is not None and not args.manifest.exists():
       from clearml import Dataset  # noqa: PLC0415

       args.manifest = (
           Path(Dataset.get(dataset_id=args.dataset_id).get_local_copy()) / "manifest.csv"
       )
   ```

   Notes:
   - Deferred import per project conventions (`# noqa: PLC0415`).
   - `Path` is already imported at the top of the file; do not re-import.
   - `Dataset.get(...).get_local_copy()` is cached, so `remap_dataset_paths()` inside
     `run_training()` will hit the same cache — no double download.

2. Remove the `return 0` after `task.execute_remotely(...)`. The current block
   (lines 344-347) is:

   ```python
   if args.enqueue:
       task.execute_remotely(queue_name=args.queue_name)
       # local process exits here; agent re-runs from top and skips this call
       return 0
   ```

   Replace with:

   ```python
   if args.enqueue:
       task.execute_remotely(queue_name=args.queue_name)
       # local process exits here via os._exit(); on agent this is a no-op,
       # falls through to run_training() below
   ```

   Rationale: `execute_remotely()` calls `os._exit()` on the local machine, so
   `return 0` is unreachable locally. On the agent, `execute_remotely()` is a
   no-op — the previous `return 0` made the agent exit with code 0 BEFORE
   `run_training()` ran. Removing it lets the agent fall through into the
   existing `try: run_training(args)` block.

Do NOT change anything else in this file. Do NOT add a docstring to `main()`.
Do NOT touch `run_training()`. Do NOT change `src/clearml_utils.py`.
