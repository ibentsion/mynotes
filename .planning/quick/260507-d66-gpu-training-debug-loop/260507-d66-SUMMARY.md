# Quick Task 260507-d66: GPU Training Debug Loop Summary

Two-bug fix that unblocks ClearML agent runs: remove early-exit after `execute_remotely()` and add manifest resolution from dataset before the path-existence check.

## What Was Done

### Task 1: Fix two bugs in src/train_ctc.py main()

**Bug 1 — Remove `return 0` after `execute_remotely()`:**

Before:
```python
if args.enqueue:
    task.execute_remotely(queue_name=args.queue_name)
    # local process exits here; agent re-runs from top and skips this call
    return 0
```

After:
```python
if args.enqueue:
    task.execute_remotely(queue_name=args.queue_name)
    # local process exits here via os._exit(); on agent this is a no-op,
    # falls through to run_training() below
```

`execute_remotely()` calls `os._exit(0)` on the local machine, making `return 0`
unreachable locally. On the agent, `execute_remotely()` is a no-op — the `return 0`
was making the agent exit with code 0 before `run_training()` ran.

**Bug 2 — Resolve manifest from ClearML dataset when not available locally:**

Inserted after `task = init_task(...)`, before the manifest existence check:

```python
# Resolve manifest from dataset when not available locally (remote agent path)
if args.dataset_id is not None and not args.manifest.exists():
    from clearml import Dataset  # noqa: PLC0415

    args.manifest = (
        Path(Dataset.get(dataset_id=args.dataset_id).get_local_copy()) / "manifest.csv"
    )
```

On the agent, `data/manifest.csv` doesn't exist. This block downloads the dataset
and sets `args.manifest` to the local copy path before the existence check. Reuses
the same cached dataset that `remap_dataset_paths()` uses inside `run_training()`.
